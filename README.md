# Автопостер VK

Telegram-бот, который **сам** публикует видео из базы парсера в VK-группы по
расписанию. Пользователь только настраивает — вручную ничего выкладывать не нужно.

```
Парсер (проект «парсер»)  →  SQLite со ссылками  →  Автопостер
   собирает ссылки            (videos: url,          читает новые ссылки,
                               duration, title)      качает, публикует в VK
                                                      по правилам и слотам
```

## Ключевая модель: правила «источник → группа»

Связь источников и групп — **многие-ко-многим** через *правила*. Правило = пара
(источник → группа) со своими настройками:

- сколько видео в день,
- слоты времени (часы МСК),
- описание к записи,
- фильтр по длительности (сек),
- порядок (старые→новые / новые→старые),
- вкл/выкл.

Один источник можно лить в несколько групп (в каждой — свои настройки), в одну
группу можно собирать несколько источников. Дедуп — на каждое правило: одно
видео уходит в одну группу ровно один раз.

## Установка

```bash
pip install -r requirements.txt   # нужен также ffmpeg в системе
cp .env.example .env               # заполни TELEGRAM_TOKEN и ADMIN_IDS
python bot.py
```

Или через Docker:

```bash
docker compose up -d --build
```

> ⚠️ Бот читает базы парсера **по путям**, указанным в источниках. В Docker
> смонтируй папку с базами парсера в контейнер (см. комментарий в
> `docker-compose.yml`) — иначе бот их не найдёт.

## Настройка через бота

1. **🔑 Токен** — задать VK-токен (инструкция через Kate Mobile прямо в боте).
2. **👥 Группы** — добавить целевые группы (по ссылке `vk.com/club123` / короткому
   имени — ID и название определяются автоматически).
3. **🎬 Источники** — добавить источник одного из двух типов:
   - **🎵 TikTok-аккаунт** — присылаешь ссылку/ник, бот сам парсит его в свою
     базу и до-парсивает раз в сутки (плюс кнопка «🔄 Обновить» вручную);
   - **🗄 Готовая база** — путь к `.db` парсера, опционально ник для фильтра.
4. **📋 Правила** — связать источник с группой и настроить: видео/день, слоты,
   описание, фильтр длительности, порядок. Включить.
5. **📊 Статус** — по каждому правилу: опубликовано / в очереди / осталось.

Дальше публикация идёт сама.

## Как работает планировщик

- Раз в сутки (`DAILY_PLAN_HOUR:MINUTE` МСК) и при старте бот сначала **до-парсивает
  источники-аккаунты** (собирает свежие ссылки в их базы), затем раскладывает план:
  для каждого включённого правила берёт до N новых видео из базы источника и
  ставит их на будущие слоты сегодня (с джиттером `PUBLISH_JITTER_SECONDS`).
- **Дневная квота**: за день по правилу планируется не более N видео —
  повторный прогон (старт + ежедневный job) не задваивает.
- В момент слота: скачивание по ссылке (yt-dlp/прямые методы) → H.264 →
  публикация в VK (`video.save` → загрузка → `wall.post`, откат при ошибке) →
  пометка «опубликовано».
- Публикация устойчива: семафор на пользователя (лимиты VK по токену) + ретраи
  с экспоненциальным backoff на временные ошибки/сеть.
- Очередь хранится в БД (`scheduled_posts`) и **переживает рестарт**.

## Переменные окружения

| Переменная | По умолчанию | Значение |
|---|---|---|
| `TELEGRAM_TOKEN` | — | токен бота (обязательно) |
| `ADMIN_IDS` | — | ID админов через запятую (видят все ошибки) |
| `VK_PUBLISH_CONCURRENCY` | 1 | одновременных публикаций на пользователя |
| `VK_PUBLISH_RETRIES` | 3 | попыток публикации при временных ошибках |
| `VK_RETRY_BASE_DELAY` | 3 | базовая задержка ретрая, сек (экспоненц.) |
| `PUBLISH_JITTER_SECONDS` | 300 | случайный сдвиг времени публикации вперёд |
| `DAILY_PLAN_HOUR` / `DAILY_PLAN_MINUTE` | 0 / 5 | когда раскладывать план на день (МСК) |
| `ERROR_RETENTION_DAYS` | 5 | сколько хранить логи ошибок |
| `ERROR_CLEANUP_INTERVAL_DAYS` | 5 | как часто чистить логи |

## Структура

| Файл | Роль |
|---|---|
| `bot.py` | Telegram-UI: токен, группы, источники, правила, статус, /errors |
| `scheduler.py` | планировщик: до-парсинг, раскладка по слотам, публикация, дедуп |
| `account_source.py` | источник-аккаунт: парсит TikTok в базу бота (обёртка парсера) |
| `parser_tiktok.py` | сам парсер TikTok (перенесён из проекта «парсер») |
| `source_reader.py` | чтение видео из базы источника (read-only) |
| `vk.py` | публикация в VK + резолв групп |
| `downloader.py` | скачивание с TikTok/Likee/YouTube/VK + H.264 (из ручного бота) |
| `db.py` | SQLite: users / groups / sources / rules / published / scheduled_posts / errors |

## Команды

`/start` · `/status` · `/rules` · `/groups` · `/sources` · `/settoken` · `/errors` · `/admin` · `/cancel`

## Источники-аккаунты (парсер встроен)

Источник типа «TikTok-аккаунт» бот парсит сам: при добавлении делает первый
прогон, дальше до-парсивает раз в сутки (перед раскладкой плана) и по кнопке
«🔄 Обновить». База такого источника лежит в `data/sources/<id>.db`. Готовые
базы парсера тоже поддерживаются (тип «🗄 Готовая база»).

## Ограничения

- Максимальная длительность ролика при скачивании — 180 сек (`downloader.py`).
- Парсер собирает только TikTok-аккаунты (как исходный проект «парсер»).
